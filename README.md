# لوحة تحكم NEXORA — نظام حقيقي يعمل بقاعدة بيانات فعلية

هذه لوحة تحكم إدارية **فعلية وليست نموذجًا مرئيًا فقط**: خادم Node.js حقيقي (Express) مع قاعدة بيانات حقيقية تُخزَّن على القرص (`server/data/db.json`)، ومصادقة حقيقية بكلمة مرور مشفّرة و JWT، وكل عمليات الإضافة/التعديل/الحذف تُحفظ فعليًا وتبقى بعد إعادة التشغيل.

## المتطلبات

- [Node.js](https://nodejs.org) الإصدار 18 أو أحدث (تحقق بالأمر `node -v`)

## التشغيل خطوة بخطوة

افتح الطرفية (Terminal) داخل مجلد `server` ونفّذ الأوامر التالية بالترتيب:

```bash
cd server
npm install       # تثبيت الحزم (مرة واحدة فقط)
npm run seed       # إنشاء قاعدة البيانات وتعبئتها ببيانات تجريبية (منتجات/طلبات/عملاء)
npm start          # تشغيل الخادم
```

بعد ظهور الرسالة `NEXORA dashboard server running: http://localhost:4000`، افتح المتصفح على:

**http://localhost:4000**

## ما تحتويه اللوحة

- **نظرة عامة**: إجمالي المبيعات، عدد الطلبات، العملاء، تنبيهات المخزون المنخفض، رسم بياني للمبيعات آخر 14 يومًا، توزيع المبيعات حسب الفئة، وأحدث الطلبات — كل الأرقام محسوبة فعليًا من قاعدة البيانات.
- **المنتجات**: إضافة / تعديل / حذف منتجات فعليًا (اسم، فئة، سعر، مخزون، حالة)، بحث وفلترة حسب الفئة، وتنبيه تلقائي عند انخفاض المخزون.
- **الطلبات**: عرض كل الطلبات وتحديث حالتها (قيد المعالجة / قيد الشحن / تم التسليم / ملغي) فعليًا عبر القائمة المنسدلة.
- **العملاء**: قائمة العملاء مع عدد طلبات كل عميل وإجمالي إنفاقه (محسوبة تلقائيًا من الطلبات الفعلية).
- **سجل الأتمتة**: سجل أحداث حي يسجّل كل عملية (إضافة منتج، تعديل، تغيير حالة طلب...) بالإضافة لزر "تشغيل الأتمتة الآن" الذي يحاكي المهام التلقائية الموصوفة في خطة العمل (تنبيهات المخزون المنخفض، تذكير السلة المتروكة، التقارير الدورية).

## البنية التقنية

```
server/                  ← الخادم الخلفي (Express + lowdb)
  src/
    index.js             ← نقطة تشغيل الخادم
    db.js                ← إعداد قاعدة البيانات
    seed.js               ← بيانات تجريبية أولية
    middleware/auth.js   ← التحقق من تسجيل الدخول (JWT)
    routes/               ← نقاط API: auth, products, orders, customers, stats, activity
  data/db.json            ← قاعدة البيانات الفعلية (تُنشأ بعد npm run seed)
public/                   ← واجهة لوحة التحكم (HTML/CSS/JS عادي، بدون أي إطار عمل خارجي)
  login.html
  dashboard.html
  assets/
```

قاعدة البيانات الحالية (`lowdb`) هي قاعدة بيانات حقيقية تُخزَّن كملف JSON على القرص — مناسبة تمامًا للتجربة والاستخدام الأولي. عند الانتقال للإنتاج الفعلي وربط اللوحة بتطبيق الجوال والموقع الحي، يُنصح بالترقية إلى PostgreSQL كما ورد في خطة العمل، وهو تغيير يقتصر على ملف `server/src/db.js` فقط دون التأثير على بقية الكود.

## لتشغيل اللوحة بشكل دائم على الإنترنت (اختياري)

اللوحة تعمل الآن محليًا على جهازك. لجعلها متاحة بشكل دائم لفريقك عبر الإنترنت، يمكن رفع مجلد `server` (وبداخله `public`) إلى إحدى منصات الاستضافة مثل Render أو Railway أو أي خادم افتراضي (VPS)، مع ضبط متغير البيئة `JWT_SECRET` بقيمة سرية خاصة بك بدلًا من القيمة الافتراضية في الكود.
